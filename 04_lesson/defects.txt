БАГ-РЕПОРТ

ID: STRUTILS-002
Заголовок: Метод contains не выбрасывает исключение при передаче некорректных типов данных

Среда выполнения:
- Python: 3.8
- ОС: Windows
- Компонент: StringUtils.contains()

Предусловие:
Импортирован класс StringUtils, создан экземпляр класса.

Шаги для воспроизведения:
1. Создать экземпляр класса StringUtils: utils = StringUtils()
2. Вызвать метод contains с параметрами:
   - string = [] (пустой список)
   - symbol = "S"
3. Ожидается исключение (TypeError или AttributeError)

Фактический результат:
Исключение не выбрасывается

Ожидаемый результат:
Выбрасывается исключение TypeError или AttributeError

Дополнительная информация:
- Тест ожидает, что при передаче нестроковых типов будет выброшено исключение
- Проблема возникает при передаче списка, словаря или других нестроковых типов

Приоритет: Средний
Серьезность: Значительная

Прикрепленные файлы:
- test_string_utils.py (тест test_contains_invalid_types)
- string_utils.py (метод contains)

Код для проверки (можно добавить в test_string_utils.py для отладки):

def test_debug_delete_symbol():
    """Отладочный тест для STRUTILS-001"""
    utils = StringUtils()
    result = utils.delete_symbol("abacaba", "ab")
    print(f"\nРезультат: '{result}'")
    print(f"Ожидание: 'acaba'")
    print(f"Совпадение: {result == 'acaba'}")
    assert result == "acaba", f"Получено '{result}', ожидалось 'acaba'"
